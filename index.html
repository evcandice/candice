Candice

Most AI assistants answer questions. Candice knows your family. She lives in iMessage, remembers what matters and shows up before you even have to ask.

Architecture

System Overview

Candice runs quietly in the background on a Mac at home. When a family member sends a message, it reads the context, pulls up what it knows about that person, asks Claude and sends a reply. The whole thing takes under two seconds.

Pipeline

Message Flow

Before Candice even talks to Claude, every message gets enriched. Photos get described, voice memos get transcribed and URLs get their titles fetched. It all happens on the home Mac. Nothing leaves the network.

01

Poll chat.db

Candice reads Apple's Messages database every 2 seconds. It only picks up messages from family members it knows about. Reactions like hearts and thumbs up are quietly ignored.

02

Rich Media Enrichment

Photos and video frames get sent to a local vision model for a description. Voice memos get transcribed by Whisper. Links get their page title fetched. All of this gets added to the message before Claude sees it.

03

Agentic Task Detection

If the message sounds like a task (fix this, build that, deploy, test), Candice spins up a Claude Code subprocess to handle it. It sends progress updates every 15 minutes and times out after 30.

04

Persona Selection

Each family member gets their own version of Candice. The dad gets direct and technical. The daughter gets patient and age-appropriate. These personality files live in butler/prompts/ and load with every message.

05

Memory Lookup + Inject

Before calling Claude, Candice searches that person's memory store for anything relevant from past conversations. Those memories get added to the prompt so Claude already knows who it is talking to.

06

Claude + Reply + Memory Write

Claude generates the reply and it gets sent back through iMessage via osascript. Then the whole conversation gets saved to memory so Candice remembers it next time.

Memory System

Long-Term Memory

Candice remembers things. Every conversation gets processed into facts, preferences and patterns. Each family member has their own private memory store that never mixes with anyone else's.

🧠

Embeddings

The nomic-embed-text model runs locally through Ollama. All the embedding work happens on the home Mac with no data sent anywhere.

⚗️

Extraction LLM

qwen3:4b runs locally through Ollama and pulls out facts and preferences from each conversation. Thinking mode is turned off because it breaks mem0's output format.

🗄️

Vector Store

Chroma stores all the family memories on disk. Each person has their own namespace so their data stays completely separate from everyone else's.

✂️

Memory Audit

A scheduled script runs regularly to clean up old or conflicting memories. Candice should only remember what is still true.

Proactive Engine

Triggers & Automation

Candice does not wait to be asked. A background loop checks every 60 seconds whether it should reach out. To keep things sane there are multiple layers of protection against over-messaging: cooldowns, mutes and a 30-minute gap between any two sends.
